== Beschreibung

MusicVoting ist eine interaktive Webanwendung, die Partygästen ermöglicht, die Musikauswahl aktiv mitzugestalten.
Gäste treten **anonym** per QR-Code oder 5-stelligem PIN bei, schlagen Songs vor und bestimmen über ein **Voting-System** (Likes) die Reihenfolge der Warteschlange.
Auf einem Monitor/TV läuft ein **Dashboard**, das die Musik im Browser abspielt und live anzeigt, was gerade läuft und was als Nächstes dran ist.
Die Wiedergabe erfolgt über den **Spotify-Premium-Account** des Gastgebers.

=== Funktionsumfang

* **Pro Party eine Warteschlange**, sortiert nach Likes (mehr Likes zuerst, bei Gleichstand ältester Wunsch zuerst).
* **Genau ein Anbieter pro Party** – Spotify (implementiert) oder YouTube (vorgesehen); nach dem Erstellen fix.
* **Anonyme Gäste** mit persistenter `deviceId`; ein Like pro Gast und Song (togglebar).
* **Host-Steuerung** (Play/Pause/Skip, Songs entfernen, Blacklist, Party beenden) – abgesichert über einen Host-PIN.
* **Live-Updates** über Server-Sent Events (Queue, Votes, aktueller Track, Fortschritt, Party-Ende).
* Optionale **iOS-App** (SwiftUI) als native Alternative für Gast und Host.

== Starten des Programms

=== Konfiguration der Anwendung

==== Spotify-Zugangsdaten hinterlegen
Die `Client ID`, das `Client Secret` und die `Redirect URI` müssen in der Konfigurationsdatei des Backends eingetragen werden.

*Datei:* `musicvoting/backend/src/main/resources/application.properties`

[NOTE]
====
Falls Sie noch keine eigenen Zugangsdaten haben,
müssen Sie ein Spotify Developer Konto unter
link:https://developer.spotify.com/dashboard/applications[developer.spotify.com] anlegen.
====

[TIP]
.Wie erstelle ich ein Spotify Developer Konto?
Siehe unter Projektübernahme!


[source,properties]
----
app.public.host=127.0.0.1
app.public.port=8080
app.web.port=4200
spotify.client.id=client_id_here
spotify.client.secret=client_secret_here
spotify.redirect.uri=http://${app.public.host}:${app.public.port}/api/spotify/callback
spotify.web.redirect.uri=http://${app.public.host}:${app.web.port}/dashboard
spotify.ios.redirect.uri=musicvotingapp://callback
quarkus.http.host=0.0.0.0
quarkus.http.port=${app.public.port}
quarkus.datasource.db-kind=postgresql
quarkus.datasource.username=musicvoting
quarkus.datasource.password=musicvoting
quarkus.datasource.jdbc.url=jdbc:postgresql://localhost:5432/musicvoting
quarkus.hibernate-orm.database.generation=none
musicvoting.join.base-url=http://localhost:4200/join
----

=== Schnellstart (alles auf einmal)

Das Skript startet Datenbank, Backend und Frontend zusammen:

[source,bash]
----
cd script
./start.sh
----

=== Komponenten einzeln starten

==== Datenbank (PostgreSQL via Docker)

Die Datenbank läuft über Docker Compose (DB: `musicvoting`, User/Passwort: `musicvoting`, Port `5432`). Das Schema wird beim ersten Start aus `musicvoting/backend/setup.sql` initialisiert.

[source,bash]
----
cd musicvoting
docker compose up -d
----

==== Backend starten

Um das Quarkus-Backend im Entwicklungsmodus zu starten:

. Navigieren Sie in das Backend-Verzeichnis:
+
[source,bash]
----
cd musicvoting/backend
----

. Starten Sie Quarkus über den Maven-Wrapper:
+
[source,bash]
----
./mvnw quarkus:dev
----

==== Frontend starten

Sobald das Backend läuft, kann die Angular-Anwendung gestartet werden:

. Navigieren Sie in das Frontend-Verzeichnis:
+
[source,bash]
----
cd musicvoting/frontend
----

. Installieren Sie die notwendigen Abhängigkeiten:
+
[source,bash]
----
npm install
----

. Starten Sie den Angular-Entwicklungsserver:
+
[source,bash]
----
ng serve --host 0.0.0.0 --port 4200
----

[IMPORTANT]
.Hinweis zum Netzwerkzugriff
Der Parameter `--host 0.0.0.0` sorgt dafür, dass die Anwendung über alle Netzwerkschnittstellen erreichbar ist. Dies ist besonders nützlich für Tests mit Mobilgeräten im selben Netzwerk oder in Container-Umgebungen, um Probleme mit der Spotify Redirect URI zu vermeiden. Dieser Modus ist ausschließlich für die *lokale Entwicklung* vorgesehen.

== Projektübernahme
Falls Sie dieses Projekt übernehmen möchten, folgen Sie bitte diesen Schritten:

=== Voraussetzungen

* *Premium Spotify Account*
** Wird benötigt, um die Spotify Playback API nutzen zu können.
* *Spotify Developer Konto*
* *Docker* (für die PostgreSQL-Datenbank)

=== Allgemeine Entwicklungshinweise

==== Nutzung der Spotify API mit einem Developer Account
Mit einem Developer Account ist die Nutzung der Applikation initial auf **25 vorgemerkte Spotify-Nutzer** beschränkt.

[IMPORTANT]
.Nutzerregistrierung
Diese User müssen manuell mit ihrer E-Mail-Adresse im Spotify Developer Dashboard zu einer Liste hinzugefügt werden.

=== Erstellen eines Spotify Developer Kontos
Falls noch kein Konto vorhanden ist, folgen Sie diesen Schritten:

. Gehen Sie zu link:https://developer.spotify.com/[Spotify for Developers].
. Melden Sie sich mit Ihrem Spotify-Konto an oder erstellen Sie ein neues.
. Klicken Sie auf *Create an App*.
. Geben Sie einen Namen und eine Beschreibung für Ihre Anwendung ein.
. Akzeptieren Sie die Nutzungsbedingungen und klicken Sie auf *Create*.

=== Applikation konfigurieren
1. Navigieren Sie zu Ihrem *Spotify Developer Dashboard*.
2. Wählen Sie die Anwendung aus, die Sie mit MusicVoting verbinden möchten.
3. Notieren Sie sich die `Client ID` und das `Client Secret`.
4. Fügen Sie die folgenden Redirect URIs in die Liste der *Redirect URIs* ein:
+
* `http://127.0.0.1:8080/api/spotify/callback`
* `http://127.0.0.1:4200/dashboard`
* `musicvotingapp://callback` (für die iOS-App)
5. Speichern Sie die Änderungen.
6. Tragen Sie die Zugangsdaten in der Datei `musicvoting/backend/src/main/resources/application.properties` ein:

[source,properties]
----
spotify.client.id=your_client_id
spotify.client.secret=your_client_secret
----

NOTE: Ersetzen Sie `your_client_id` und `your_client_secret` durch die tatsächlichen Werte aus Ihrem Dashboard.

=== Veröffentlichen der Applikation
Um die Anwendung offiziell zu veröffentlichen, muss ein **Antrag bei Spotify** gestellt werden. Nach der Genehmigung ist die Anwendung für alle Spotify-Nutzer zugänglich, und die Beschränkung auf 25 Nutzer entfällt.
